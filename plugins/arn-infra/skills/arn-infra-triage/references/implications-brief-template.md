# Infrastructure Implications Brief Template

Template for the structured implications brief produced by the `arn-infra-request-analyzer` agent during triage. This brief is consumed by `arn-infra-define` and the wizard to generate IaC configurations.

---

## Template

```markdown

**Source:** [Issue reference or assessment ID]
**Analyzed:** [ISO 8601 timestamp]
**Analyzer:** arn-infra-request-analyzer

---
```

---

## Usage

This brief is:

1. **Produced by** the `arn-infra-request-analyzer` agent during `arn-infra-triage`
2. **Presented to** the user for review and approval
3. **Consumed by** `arn-infra-define` to generate actual IaC configurations
4. **Consumed by** the `arn-infra-wizard` for end-to-end infrastructure setup
5. **Optionally saved** as `INFRA_<feature-name>.md` in the specs directory for later reference

---

## Notes

- Cost estimates are ranges, not exact values. Usage-based pricing introduces variability.
- The "Source" field links back to the original infrastructure request issue for traceability.
- Provider service names are adapted to the user's configured providers (e.g., AWS RDS vs. Fly Postgres).
- The "Recommended Approach" section provides actionable guidance that `arn-infra-define` can follow directly.
- For full application assessments (Mode B of the analyzer), multiple briefs are produced -- one per priority category (foundation, core, enhancement).
